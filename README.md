## 📱 Telegram Relational Database Case Study

### 📌 Project Overview

This project presents a relational database design for **Telegram**, one of the world's leading cloud-based messaging platforms. The database schema models Telegram's core functionalities, including user management, chats, messages, channels, groups, bots, files, and reactions.

The project demonstrates the process of transforming real-world application requirements into a normalized relational database using Entity Relationship (ER) modeling and relational schema design.



### 🎯 Objectives

* Design a scalable relational database for Telegram.
* Identify entities, attributes, and relationships.
* Apply normalization principles to reduce data redundancy.
* Define Primary Keys and Foreign Keys.
* Create an Entity Relationship (ER) Diagram.
* Convert the ER model into a relational schema.
